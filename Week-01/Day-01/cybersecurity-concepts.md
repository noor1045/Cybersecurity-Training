# Basic Cybersecurity Concepts
## 1. Vulnerability Assessment (VA)
Vulnerability Assessment is the process of checking a computer system, network, or application for security weaknesses.
The main purpose of a vulnerability assessment is to find problems before an attacker can use them. These problems may include outdated software, open ports, weak passwords, missing updates, or incorrect security settings.
Some commonly used tools are:
- Nessus
- OpenVAS
- Nmap
- Qualys
For example, a company may scan its network and discover that one of its servers is running an old version of software that contains known security issues.

## 2. Penetration Testing (PT)
Penetration Testing is an authorized security test in which a tester tries to find and safely exploit weaknesses in a system.
The purpose of penetration testing is to understand how serious a vulnerability is and what an attacker could do if the weakness was exploited.
A penetration test normally includes:
1. Collecting information about the target
2. Scanning the system
3. Finding possible vulnerabilities
4. Testing selected vulnerabilities
5. Recording the results
6. Writing a final report

Some common penetration testing tools are:
- Metasploit
- Burp Suite
- Nmap
- Wireshark
- SQLmap
For example, a penetration tester may check whether a weak password or an insecure website form can be used to gain unauthorized access.

### Difference Between VA and PT
Vulnerability Assessment mainly focuses on finding and reporting security weaknesses.
Penetration Testing goes one step further by checking whether those weaknesses can actually be exploited.
In simple words:
- VA tells us what weaknesses exist.
- PT tells us what could happen if those weaknesses are attacked.

## 3. Security Operations Center (SOC)
A Security Operations Center is a team that monitors an organization's systems and networks for security threats.
SOC analysts check alerts, review logs, investigate suspicious activity, and respond to security incidents.
The main responsibilities of a SOC include:
- Monitoring security alerts
- Detecting unusual activity
- Investigating possible attacks
- Analyzing system and network logs
- Responding to incidents
- Preparing security reports
Some common SOC tools are:
- Splunk
- Wazuh
- Microsoft Sentinel
- IBM QRadar

For example, if a user account shows many failed login attempts, the SOC team may investigate whether someone is trying to break into the account.

## 4. Incident Response (IR)
Incident Response is the process of dealing with a cybersecurity incident in a controlled and organized way.
The goal is to limit the damage, remove the threat, restore the affected systems, and prevent the same incident from happening again.
The main phases of Incident Response are:
1. Preparation
2. Identification
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

### Preparation
The organization prepares its team, tools, policies, and response procedures.

### Identification
The security team checks whether a real security incident has occurred.

### Containment
The affected device or system is isolated so the incident does not spread.

### Eradication
The threat, malware, unauthorized account, or root cause is removed.

### Recovery
The affected systems are restored and monitored before returning to normal use.

### Lessons Learned
The team reviews the incident and identifies improvements for the future.
For example, if malware is detected on a computer, the device may first be disconnected from the network. The malware is then removed, the system is restored, and the cause of the infection is investigated.

## 5. Digital Forensics
Digital Forensics is the process of collecting, preserving, and analyzing digital evidence.
It is commonly used to investigate cyberattacks, data theft, unauthorized access, fraud, and other digital crimes.
Digital Forensics may include:
- Computer forensics
- Network forensics
- Mobile forensics
- Memory forensics
- Cloud forensics
The basic process usually includes:
1. Identifying the evidence
2. Preserving the original data
3. Collecting relevant information
4. Examining the evidence
5. Analyzing the findings
6. Preparing a report
Some common digital forensics tools are:
- Autopsy
- FTK Imager
- EnCase
- Volatility
- Wireshark

For example, a forensic investigator may examine deleted files, browser history, system logs, emails, or network traffic to understand what happened during an incident.
